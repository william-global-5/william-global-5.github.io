The FDOT Traffic Engineering and Operations Office keeps Florida safely on the go. For example, the ITSFM software enables accurate data and camera feeds to the statewide 511 Traffic Information System, used by millions of people each year. ITSFM helps keep FDOTâ€™s real-time information accurate.

The following steps discuss the process for populating Feature Import Templates.

Step 1

Download the latest version of the appropriate Feature Import Template to your computer from this website. The ITSFM is routinely updated. Therefore, ensure you are working from the most current templates available.

Note: For returning users, please note the Master Code List (MCL) will automatically download each time you open an ITSFM Import Template.

Step 2

Compile a list of features installed by the project then compare your list against the feature types and attributes currently supported in the Feature Import Templates. The import templates dropdown menus will not display feature types that were not previously setup in the Master Code Lists (MCL). The MCL is password protected to restrict user edits. The FDOT Central Office manages the MCL to ensure the code list is consistent, accurate, up-to-date and available to user.

To request MCL updates, email a list of new features including all support attribution and manufacturer cut sheets to import-templates@fdot.gov. The Central Office will:

  • Update the MCL,
  • Updated this website with the latest version, and
  • Notify you the update has been completed within two business days.

Step 3

Rename and save the Import Template with a new name that is meaningful and specific to your project. For example, replace the template number with your project number and feature type as shown in the following example:

(ITSFM5006) - Equipment Cabinets & Support Structures Import Template
123456789 - Equipment Cabinets & Support Structures Import Template

Step 4

Open the appropriate Import Template file using the most current version of MS Excel. The Master Code List will download automatically when you open the file and must remain open at all times while populating the template. A reload button is available within the first cell labeled MCL which will force a reload/download of the MCL.

Step 5

Input feature and attributes into the Import template including ALL required fields denoted in the column header for each workbook. The ITSFM system will abort the importing process if any required field is missing.

Step 6

Validate the completeness and accuracy of the populated data then forward the file to the project engineer for testing and acceptance.
